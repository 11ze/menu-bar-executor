# menu-bar-executor 应用优化增强计划

## Context

macOS 菜单栏命令执行器需要全面优化。经过代码审查，发现了多个代码质量问题、架构缺陷和功能缺失。本计划将分四个阶段实施：代码质量修复 → 架构优化 → 功能增强 → 测试保障。

**预期成果**：
- 修复所有已知 Bug 和代码质量问题
- 提升数据安全性和用户体验
- 添加命令分组、快捷键、搜索、执行历史功能
- 建立单元测试体系

---

## Phase 1: 代码质量修复

### 1.1 Command 模型重构（引入 UUID）

**问题**：当前使用 `name` 作为 ID，导致无法重命名命令。

**文件**：[Command.swift](Sources/App/Command.swift)

**改动**：
```swift
struct Command: Identifiable, Codable {
    let id: UUID                    // 新增：独立唯一标识
    var name: String                // 改为 var，允许修改
    let command: String
    var workingDirectory: String?
    var icon: String?
    var notification: Bool
    var group: String?              // Phase 3 预留：分组功能
    var shortcut: String?           // Phase 3 预留：快捷键

    init(id: UUID = UUID(), name: String, ...) { ... }
}
```

**兼容性**：添加 JSON 解码兼容逻辑，处理旧配置文件。

### 1.2 Bug 修复

**NotificationManager.showFailure 重复 output** ([NotificationManager.swift:48-52](Sources/App/NotificationManager.swift#L48-L52))

```swift
// 修复前
var body = "command: \(commandName)\noutput: \(error)"
body += "\noutput: \(truncatedOutput)"  // 重复

// 修复后
var body = "command: \(commandName)\nerror: \(error)"
if let output = output, !output.isEmpty {
    body += "\noutput: \(truncatedOutput)"
}
```

## Phase 2: 架构优化

### 2.1 错误处理机制

**新增**：[AppError.swift](Sources/App/AppError.swift)
```swift
enum AppError: LocalizedError {
    case configLoadFailed(Error)
    case configSaveFailed(Error)
    case commandTimeout(seconds: Int)
    case commandExecutionFailed(String)
}
```

**修改**：CommandsManager 添加 `@Published var lastError: AppError?`

**修改**：视图层添加 `.alert` 错误提示

### 2.2 数据一致性保证

**原子写入** ([ConfigLoader.swift](Sources/App/ConfigLoader.swift))：
```swift
func saveConfig(_ commands: [Command]) throws {
    let tempPath = configFilePath.appendingPathExtension("tmp")
    try data.write(to: tempPath)
    try FileManager.default.replaceItem(at: configFilePath, withItemAt: tempPath)
}
```

**操作回滚** ([CommandsManager.swift](Sources/App/CommandsManager.swift))：
```swift
func deleteCommand(id: UUID) {
    guard let index = commands.firstIndex(where: { $0.id == id }) else { return }
    let backup = commands[index]
    commands.remove(at: index)
    do {
        try configLoader.saveConfig(commands)
    } catch {
        commands.insert(backup, at: index)  // 回滚
        lastError = .configSaveFailed(error)
    }
}
```

### 2.3 CommandExecutor 超时机制

## Phase 3: 功能增强

### 3.1 命令分组

**数据模型**：Command 添加 `group: String?` 字段

**菜单逻辑** ([AppDelegate.swift](Sources/App/AppDelegate.swift))：
- 无分组命令 → 顶层菜单项
- 有分组命令 → 子菜单

**UI** ([CommandEditorView.swift](Sources/App/Views/CommandEditorView.swift))：
- 添加分组选择器（支持新建分组）

### 3.2 快捷键绑定

**数据模型**：Command 添加 `shortcut: String?` 字段

**实现方案**：使用 `NSEvent.addGlobalMonitorForEvents` 监听全局快捷键

**UI**：编辑器添加快捷键录制组件

### 3.3 命令搜索

**位置**：菜单顶部添加搜索框

**实现**：实时过滤命令列表

### 3.4 执行历史

**新增**：[ExecutionHistory.swift](Sources/App/Services/ExecutionHistory.swift)

**存储**：`~/.config/menu-bar-executor/history.json`（最近 100 条）

**UI**：新增历史窗口，显示命令名称、时间、输出
