# 修复菜单栏执行器通知功能

## 背景

用户报告命令执行后不再显示系统通知。经分析，根因是 `NotificationManager.swift` 缺少 `UNUserNotificationCenterDelegate` 实现。

macOS 的 UNUserNotificationCenter 默认行为：当 App 在前台运行时，不会显示通知横幅。必须设置 delegate 并实现 `userNotificationCenter(_:willPresent:withCompletionHandler:)` 方法，显式调用 `completionHandler([.banner, .sound])` 才能显示前台通知。

## 修改文件

### 1. NotificationManager.swift

**路径**: `Sources/App/NotificationManager.swift`

**修改内容**:

1. 让 `NotificationManager` 遵循 `UNUserNotificationCenterDelegate` 协议
2. 在 `init()` 中设置 delegate：`UNUserNotificationCenter.current().delegate = self`
3. 实现 delegate 方法：

```swift
nonisolated func userNotificationCenter(
    _ center: UNUserNotificationCenter,
    willPresent notification: UNNotification,
    withCompletionHandler completionHandler: @escaping (UNNotificationPresentationOptions) -> Void
) {
    completionHandler([.banner, .sound])
}
```

**注意**：使用 `nonisolated` 因为 delegate 方法不在 MainActor 上下文中调用。

### 2. CommandExecutor.swift（可选优化）

**路径**: `Sources/App/CommandExecutor.swift`

**修改内容**：支持自动 source shell 配置文件

当前命令执行方式：
```swift
process.arguments = ["-c", command.command]
```

修改为：
```swift
process.arguments = ["-l", "-c", command.command]
```

**说明**：`-l` (login shell) 会自动 source `~/.zprofile`、`~/.zshrc` 等配置文件，使命令可以使用用户定义的环境变量和别名。

**不采用** `source ~/.zshrc && command` 的方式，因为：
- 如果用户的 `.zshrc` 有输出会污染命令结果
- `zsh -l` 是更标准的做法

## 验证步骤

1. 构建并运行应用：
   ```bash
   pkill -f menu-bar-executor && xcodebuild -project menu-bar-executor.xcodeproj -scheme menu-bar-executor build && open ./build/Build/Products/Debug/menu-bar-executor.app
   ```

2. 点击任意命令执行

3. 确认系统通知横幅正常显示

4. 检查通知内容是否包含命令名称和输出

## 风险评估

- **低风险**：修改仅涉及通知显示逻辑，不影响命令执行功能
- **兼容性**：`UNUserNotificationCenterDelegate` 是标准 API，支持 macOS 10.14+
