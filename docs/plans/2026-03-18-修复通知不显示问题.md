# 修复通知不显示问题

## Context

用户报告重载和执行命令后没有发送通知。分析发现 `NotificationManager` 缺少 `@MainActor` 标记，导致 `UNUserNotificationCenter.add()` 可能在非主线程执行，造成通知发送失败。

## 修改文件

### Sources/App/NotificationManager.swift

**改动**：为 `NotificationManager` 类添加 `@MainActor` 标记

```swift
// 修改前
final class NotificationManager: NSObject, UNUserNotificationCenterDelegate {

// 修改后
@MainActor
final class NotificationManager: NSObject, UNUserNotificationCenterDelegate {
```

**说明**：
- `UNUserNotificationCenter` 的 API 应在主线程调用
- `CommandsManager` 是 `@MainActor`，调用 `NotificationManager` 方法时需要在同一 actor 上下文中
- 保持 `nonisolated` 标记在 `userNotificationCenter(_:willPresent:...)` delegate 方法上，因为系统调用不一定在主线程

## 验证

```bash
# 构建
xcodebuild -project menu-bar-executor.xcodeproj -scheme MenuBarExecutor -configuration Debug -derivedDataPath ./build build

# 运行
pkill -f MenuBarExecutor; open ./build/Build/Products/Debug/MenuBarExecutor.app

# 测试步骤：
# 1. 点击任意命令执行，确认通知显示
# 2. 按 Cmd+R 重载配置，确认通知显示
```

## 风险评估

- **低风险**：仅添加 actor 标记，不改变逻辑
- `nonisolated` delegate 方法保持不变，系统回调正确处理
