# 菜单栏命令执行器设计

## 1. 项目概述

**项目名称**：menu-bar-executor
**类型**：macOS 菜单栏工具应用
**核心功能**：在菜单栏显示自定义命令列表，点击后执行并显示结果通知

## 2. 技术栈

- **UI 框架**：SwiftUI Menu (macOS 12+)
- **语言**：Swift 5.9+
- **配置文件**：JSON
- **构建工具**：XcodeGen

## 3. 项目结构

```
menu-bar-executor/
├── Sources/
│   └── App/
│       ├── AppDelegate.swift           # App 入口，菜单栏配置
│       ├── StatusBarView.swift        # 菜单栏视图（SwiftUI Menu）
│       ├── CommandsManager.swift      # 命令管理 + 执行 + 重载
│       ├── ConfigLoader.swift         # 配置加载
│       └── NotificationManager.swift  # 通知管理
├── Resources/
│   ├── Assets.xcassets/
│   └── commands.json                  # 默认命令配置
├── project.yml                        # XcodeGen 配置
└── SPEC.md                            # 本设计文档
```

## 4. 配置文件

### 4.1 配置路径

- 用户配置：`~/.config/menu-bar-executor/commands.json`
- 默认配置：应用 Resources 中的 commands.json

### 4.2 配置格式

```json
{
  "commands": [
    {
      "id": "open-idea",
      "name": "打开 IntelliJ IDEA",
      "command": "open -a \"IntelliJ IDEA\"",
      "icon": "hammer.fill",
      "notification": true
    },
    {
      "id": "git-pull",
      "name": "Git Pull",
      "command": "git pull",
      "workingDirectory": "~/Projects/myapp",
      "icon": "arrow.down.circle",
      "notification": true
    }
  ]
}
```

### 4.3 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 唯一标识 |
| name | String | 是 | 菜单显示名称 |
| command | String | 是 | 要执行的命令 |
| workingDirectory | String | 否 | 执行目录，默认 ~ |
| icon | String | 否 | SF Symbol 图标，默认 terminal |
| notification | Bool | 否 | 是否显示通知，默认 true |

## 5. 核心功能

### 5.1 菜单栏

- 使用 SwiftUI `Menu` 组件
- 菜单栏图标：`terminal.fill`（SF Symbol）
- 无 Dock 图标（LSUIElement = true）

### 5.2 菜单结构

```
┌────────────────────────────┐
│ 打开 IntelliJ IDEA        │
│ Git Pull                  │
├────────────────────────────┤
│ ↻ 重载配置                │
├────────────────────────────┤
│ 退出                      │
└────────────────────────────┘
```

### 5.3 命令执行

- 使用 `Process` 执行系统默认 shell
- 异步执行，不阻塞 UI
- 执行完成后根据结果显示通知

### 5.4 通知

- 使用 `UserNotifications` 框架
- 成功：显示命令名称 + "执行成功"
- 失败：显示命令名称 + "执行失败" + 错误信息

### 5.5 重载配置

- 菜单项点击触发
- 重新读取配置文件
- 显示 "配置已重载" 通知

## 6. 数据流

```
启动 App
    ↓
ConfigLoader 加载配置
    ↓
CommandsManager 管理命令列表
    ↓
StatusBarView 显示菜单
    ↓
用户点击命令 → CommandExecutor 执行
    ↓
NotificationManager 显示结果通知
```

## 7. 错误处理

| 场景 | 处理 |
|------|------|
| 配置文件不存在 | 创建默认配置目录和文件 |
| JSON 解析错误 | 显示错误通知，跳过加载 |
| 命令执行失败 | 显示失败通知，包含错误信息 |
| 目录创建失败 | 显示错误通知 |

## 8. macOS 特性

- **无 Dock 图标**：设置 LSUIElement = true
- **后台运行**：点击命令后不激活窗口
- **系统通知**：使用 UserNotifications 框架
