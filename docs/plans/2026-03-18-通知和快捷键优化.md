# 通知和快捷键优化

## Context

用户希望优化两个功能：
1. **通知内容**：执行命令后的通知需要更好的换行格式，使内容更易读
2. **快捷键设置**：优化交互体验，添加冲突检测，避免多个命令使用相同快捷键

---

## 1. 通知内容优化

### 现状分析

当前通知格式（[NotificationManager.swift](Sources/App/NotificationManager.swift)）：
- 成功：`command: {name}\noutput: {output}` （单行紧凑）
- 失败：`command: {name}\nerror: {error}\noutput: {output}`

### 优化方案

改进通知换行格式，增加空行使内容更清晰：

**成功通知**：
```
✅ 执行成功

命令：{name}

输出：
{output}
```

**失败通知**：
```
❌ 执行失败

命令：{name}

错误：
{error}

输出：
{output}
```

### 修改文件

- [NotificationManager.swift](Sources/App/NotificationManager.swift)：修改 `showSuccess` 和 `showFailure` 方法

---

## 2. 快捷键优化

### 现状分析

当前问题（[CommandEditorView.swift](Sources/App/Views/CommandEditorView.swift)）：
- 使用简单 TextField，无验证
- 无冲突检测，多个命令可设置相同快捷键
- 用户体验差，需要手动输入字符

### 优化方案

#### 2.1 添加冲突检测

在 [CommandEditorView.swift](Sources/App/Views/CommandEditorView.swift) 中：
- 添加计算属性检查当前快捷键是否与其他命令冲突
- 冲突时显示警告信息（红色文字提示）

#### 2.2 优化 UI 交互

- 实时显示已使用的快捷键列表
- 冲突时输入框边框变红 + 显示警告图标
- 显示当前可用的快捷键提示

#### 2.3 格式验证

- 限制为单个字符（a-z, 0-9）或功能键（F1-F12）
- 自动转换为大写显示

### 修改文件

- [CommandEditorView.swift](Sources/App/Views/CommandEditorView.swift)：
  - 添加 `shortcutConflict` 计算属性
  - 修改快捷键 Section UI，显示冲突警告
  - 添加已使用快捷键列表显示

---

## 实现步骤

### Step 1: 优化通知格式

修改 `showSuccess` 和 `showFailure` 方法，使用更清晰的换行格式。

### Step 2: 添加快捷键冲突检测

1. 在 CommandEditorView 添加 `shortcutConflict` 计算属性
2. 检查是否有其他命令（排除当前编辑的命令）使用相同快捷键

### Step 3: 优化快捷键 UI

1. 冲突时显示红色警告文字
2. 显示已使用的快捷键列表
3. 添加格式验证提示

---

## 验证方法

1. 构建并运行应用
2. 执行命令，验证通知显示格式
3. 测试快捷键冲突检测：
   - 给命令 A 设置快捷键 "r"
   - 给命令 B 也设置快捷键 "r"，验证是否显示冲突警告
4. 验证列表中显示已使用的快捷键

## 关键代码变更

### NotificationManager.swift

```swift
func showSuccess(commandName: String, output: String?) {
    let content = UNMutableNotificationContent()
    content.title = "✅ 执行成功"
    content.subtitle = commandName

    if let output = output?.truncated(to: 100), !output.isEmpty {
        content.body = "输出：\n\(output)"
    }

    // ...
}

func showFailure(commandName: String, error: String, output: String?) {
    let content = UNMutableNotificationContent()
    content.title = "❌ 执行失败"
    content.subtitle = commandName

    var body = "错误：\n\(error)"
    if let output = output?.truncated(to: 100), !output.isEmpty {
        body += "\n\n输出：\n\(output)"
    }
    content.body = body

    // ...
}
```

### CommandEditorView.swift

```swift
// 检查快捷键冲突
private var shortcutConflict: Bool {
    guard !shortcut.isEmpty else { return false }
    return manager.commands.contains { cmd in
        cmd.id != command?.id && cmd.shortcut?.lowercased() == shortcut.lowercased()
    }
}

// 获取已使用的快捷键
private var usedShortcuts: [(String, String)] {
    manager.commands.compactMap { cmd in
        guard let s = cmd.shortcut, !s.isEmpty, cmd.id != command?.id else { return nil }
        return (s, cmd.name)
    }
}

// UI Section
Section("快捷键") {
    HStack {
        TextField("快捷键（如：r, 1, F1）", text: $shortcut)
            .help("输入单个字符或功能键名称")

        if shortcutConflict {
            Image(systemName: "exclamationmark.triangle.fill")
                .foregroundColor(.red)
                .help("快捷键已被其他命令使用")
        }
    }

    if shortcutConflict {
        Text("⚠️ 快捷键冲突：此快捷键已被使用")
            .font(.caption)
            .foregroundColor(.red)
    }

    if !usedShortcuts.isEmpty {
        VStack(alignment: .leading, spacing: 4) {
            Text("已使用的快捷键：")
                .font(.caption)
                .foregroundColor(.secondary)
            ForEach(usedShortcuts, id: \.0) { shortcut, name in
                Text("  ⌘\(shortcut.uppercased()) - \(name)")
                    .font(.caption2)
                    .foregroundColor(.secondary)
            }
        }
    }
}
```
